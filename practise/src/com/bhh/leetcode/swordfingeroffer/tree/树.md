关于树的许多题目都可以使用递归来实现, 关于递归
    很多是重塑树,(删除节点)
    也有一部分是返回某一个节点,(查询最大小节点)
    也有是遍历输出,(前序遍历和该问题)
    也有循环遍历是每次循环俩个节点
    + 返回值类型
        + TreeNode, 用于重塑树
        + void, 直接遍历树
        + 其他(int), 用于获取其他类型数据, 例如节点深度, 叶子结点"深度"(反深度)为1 (if node == null return 0;)
    + 注意点 : 
        + 若是出现情况需要提前退出递归, 可以返回特性值, 
          需要注意的是, 所有递归方法后都需要返回该特殊值, 否则值可能会被替代
          例如 : Question55_2, 方法若返回了-1, 则递归的方法都进行判断若是 = -1, 则return -1(都返回-1, 一路回溯回去)
    
关于树 + 递归寻找规律的方法 : 
    结束条件是叶子结点, 看如果是叶子结点应该返回什么
    循环条件是正常节点, 看root节点下的俩个节点应该干什么 
    根据这几个具有标志性的节点来寻找规律
    

遍历
    前序,中序,后续遍历
    
    中序遍历
        + 对于二分搜索树, 中序遍历是从小到大输出数据, 中序倒叙遍历是从大到小输出, 
            有时倒叙可能会更好
    后序遍历
        后序遍历是自底向上的遍历方法
        
    层序遍历
        + 层序遍历是自顶向下的遍历方式
        + 层序遍历如何分层, 在while中添加for(i = queue.size() - 1; i >= 0 ; i--),
            这个循环会输出当前层的节点数, 使用i--是因为如果使用i++queue.size()是变化的, 条件变话可能会造成死循环
    
深度问题(自底向上, 自顶向下)
    Question55_1和Question55_2俩个问题
        + Question55_1使用后序遍历, 自底向上遍历方式, 求出深度, 
          但是因为是自底向上的方式, 所以求得的是反向的深度, 例如 : 根据后序遍历根节点的深度是叶子节点的深度
           
        + Question55_2
            + 根据后续遍历 获取 节点的深度, 然后将节点之间的深度进行相减来实现判断
