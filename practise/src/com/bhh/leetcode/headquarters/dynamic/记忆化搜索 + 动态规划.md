动态规划 自底向上
+ 这个底应该怎么设置?
    观察Question198和01背包问题大概总结 : 这个底一定是要某一种极端情况下, 
    可以得到的或唯一解来直接赋值给数组
    然后根据这个底来展开
+ 动态规划 和 记忆化搜索并没有严格区分底和顶, 所以按照题意来区分较好

+ 在创建递归算法的时候, 可以从不同顺序解题, 
有些题目从前往后容易解(index 到 num.length-1), 
有些题目从后往前容易解(0 到 index)

+ 动态规划 和 记忆化搜索的转换 : 
    转换情况不一, *但是递归可以转换一层for(转换看结束条件), 若递归中还有一层for, 则这一层for可以继承到递归转换的for中*
    例如 : Question198 : 递归结束条件是 if (index > n - 1) return 0; 以索引为基, 
    则转换为 for (int i = n - 2; i >= 0; i--) (n - 1为底, 被赋值了)
    再如 : ZeroOneBackpack : 递归结束条件是 if (c <= 0 || i >= w.length) return 0; 以重量或者索引为基
    则转换为 for (int j = 0; j < c + 1; j++)
